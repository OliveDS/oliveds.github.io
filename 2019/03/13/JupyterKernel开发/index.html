JVM BaseKernel 安装

在GitHub上的源码地址👇

https://github.com/SpencerPark/jupyter-jvm-basekernel

因为后面需要对它进行修改,所以先能够把它build成功跑起来,就成功了一半(哈?give or take…🤷‍♀️)

可以使用gradle或Maven来build这个project

这里我们使用gradle(比Maven功能更加强大)

gradle是一个项目自动化建构工具.

  • 差异管理:可实现静态级的差异控制,避免了针对不同设备产生的冗余代码
  • 依赖管理:自动装载到~/.gradle/下,完成classpath等配置
  • …

Gradle通过编写一个名为build.gradle的脚本文件对项目进行设置，再根据这个脚本对项目进行构建（复杂的项目也有其他文件）

如果使用IDEA来编译这个project,那么IDEA已经自带Gradle包,没有必要重新安装了

IJAVA Kernel 源码

https://github.com/SpencerPark/IJava

👆GitHub上的地址

可以看到它还提供了一个jupyter-jvm-basekernel,解释说

If you are interested in building your own kernel that runs on the JVM check out the related project that this kernel is build on

另外这个jvm-basekernel解决的是aims to cover the messaging protocol alone部分的问题,程序员至少需要将evl部分写好,才能完成一个kernel的基本工作.

也就是说其实我开发的Kernel可以在这个基础上添加内容就可以了?下载下来和IJAVA源码对比看后,发现其实IJAVA就也是基于它做的,所以我可以两个对比,看看代码到底是怎样写的.

IJAVA vs Jupyter JVM BaseKernel

File Item IJAVA JVM Base
setting.gradle rootProject.name Ijava jupyter-jvm-basekernel
gradlew.bat Gradle startup script for Windows - -
Gradlew Gradle start up script for UN*X - -
build.gradle

原本预想对对两者进行一个比较,找出需要修改的地方,直接开始码代码,结果发现文件结构差异还是挺大的,直接开始刚代码有点困难(无从下手),所以还是先了解一下原理吧~

iPython Kernel

IPython提供了5个接口,与Jupyter前端页面进行信息交换;另外,Kernel背靠Python engine,将用户指令发给engine执行然后获取运行结果返回给用户.

(我浅显的理解,如果有误请纠正)

JVM BaseKernel

是一个提供了通信(messaging)和基本操作(还包括代码检查补全)功能的BASE(名副其实)

而example文件夹是利用这个Base实现的一个实际Kernel(Nashorn Kernel) ,其中起到功能作用的代码只有INashornJS.java和NashornKernel.java这两个文件(🤔看起来好像不难改)

INashornJS.java

主要是对文件进行检查,创建Handler,然后与Jupyter建立连接.

这个代码可以直接使用在新写的Kernel中
